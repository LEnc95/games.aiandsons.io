School Data Policy

Effective date: March 6, 2026

This page summarizes school-facing privacy expectations for Cade's Games. It is intended to support district and classroom review.

School-Safe Design Defaults

  • Students can play without creating accounts
  • No chat or public social features
  • No targeted advertising in classroom workflows
  • Classroom mode can restrict games and lock shop actions during sessions

Student Data Scope

  • This build stores gameplay state locally in browser storage
  • No centralized student roster is required for core gameplay
  • No personal student profile is required beyond optional display name

Operational Commitments

  • Minimize data collection to what is necessary for gameplay
  • Avoid behavioral targeting and unnecessary tracking
  • Document and review any future backend data collection before release
  • Update legal and privacy notices before enabling new data flows

FERPA and COPPA Readiness Notes

For school deployments, contracts and implementation should align with district FERPA requirements. If child-directed monetization or data processing changes are introduced, COPPA controls and disclosures must be updated before launch.

Change Control

Any feature that introduces remote student data collection, third-party analytics, or ad tech requires a documented privacy review and policy revision.

For general privacy details, see the Privacy Policy.
